# Content Authoring Guide

How to create and manage content for alexrosenkranz.com.

## Quick Start

### Create a New Post

Use the Claude Code command:

```
/project:new-post [description of what you're posting]
```

Or manually create a file at `src/content/posts/YYYY-MM-DD-slug.md`:

```yaml
---
title: 'Interesting Article About Web Performance'
publishedAt: 2025-01-15T10:30:00Z
tags:
  - article
url: 'https://example.com/article'
draft: false
---
Optional commentary about why this is interesting.
```

### Create a New Article

Use the Claude Code command:

```
/project:new-article [title and topic]
```

Or manually create a file at `src/content/articles/slug.md`:

```yaml
---
title: 'Building My Personal Site'
publishedAt: 2025-01-15T09:00:00Z
description: 'A walkthrough of the tech stack and decisions behind this site.'
tags:
  - web-dev
  - astro
draft: true
---
Article content goes here...
```

## Content Types

### Posts

Quick, small items you want to share. Think of these as "digital breadcrumbs."

**When to use:** Links, photos, quick recommendations, code snippets, quotes

**Tags (pick one or more):**
| Tag | Use For |
|-----|---------|
| `article` | Links to external articles |
| `photo` | Images with optional caption |
| `product` | Physical or digital products |
| `code` | Code snippets or repos |
| `music` | Albums, songs, playlists |
| `video` | YouTube, Vimeo, etc. |
| `quote` | Text quotes |
| `note` | Standalone thoughts |
| `website` | Interesting websites |
| `app` | Mobile or desktop applications |
| `movie` | Films |
| `tv` | TV shows |

**Filename format:** `YYYY-MM-DD-slug.md`

### Articles

Long-form writing that takes more time and thought.

**When to use:** Tutorials, essays, deep dives, project writeups

**Filename format:** `slug.md` (no date prefix)

### Digests

Monthly rollups of posts. Usually auto-generated but can be manually curated.

**Filename format:** `YYYY-MM.md` (e.g., `2025-MM.md`)

## Frontmatter Reference

### Posts

```yaml
---
title: string # Required
publishedAt: datetime # Required, ISO 8601
tags: string[] # Required, at least one
url: string # Optional, for link posts
image: string # Optional, relative path for photos
draft: boolean # Optional, defaults to false
---
```

### Articles

```yaml
---
title: string # Required
publishedAt: datetime # Required, ISO 8601
updatedAt: datetime # Optional
description: string # Required, 120-160 chars for SEO
tags: string[] # Optional
draft: boolean # Optional, defaults to false
---
```

### Digests

```yaml
---
title: string # Required, e.g., "<month> Digest"
publishedAt: datetime # Required
monthStart: date # Required
monthEnd: date # Required
autoGenerated: boolean # Required
featuredPosts: string[] # Optional, for manual curation
excludedPosts: string[] # Optional, posts to exclude
intro: string # Optional, monthly summary
---
```

## Writing Tips

### For Posts

- **Keep it brief.** A sentence or two of commentary is plenty.
- **Let the link speak.** For link posts, the title and URL are the content.
- **Use meaningful tags.** They help with filtering and discovery.

### For Articles

- **Write a compelling title.** Under 60 characters for SEO.
- **Nail the description.** 120-160 characters, used for social sharing.
- **Start with draft: true.** Don't publish until it's ready.
- **Use headings.** Break up long content with H2 and H3 headings.

### General

- **Dates matter.** The `publishedAt` date determines sort order.
- **Slugs are forever.** Once published, don't change the filename.
- **Preview locally.** Run `pnpm dev` to see how content looks.

## Images

### In Posts

For photo posts, add images to the same directory or a shared assets folder:

```yaml
---
title: 'My Dog'
publishedAt: 2025-01-15T10:00:00Z
tags:
  - photo
image: './dog.jpg'
---
Caption for the photo.
```

### In Articles

Use standard markdown syntax:

```markdown
![Alt text describing the image](./images/screenshot.png)
```

Images are automatically optimized by Astro's image pipeline.

## Publishing

1. Set `draft: false` in frontmatter
2. Commit and push to `main` branch
3. Cloudflare Pages deploys automatically

## Common Issues

### "Content validation failed"

Check that:

- All required frontmatter fields are present
- Tags are from the valid list
- Dates are valid ISO 8601 format

### "Image not found"

Check that:

- Image path is relative to the content file
- File extension matches exactly (case-sensitive)

### Content not showing up

Check that:

- `draft: false` is set
- `publishedAt` date is not in the future
- You've saved the file
