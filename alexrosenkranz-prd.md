# Product Requirements Document: alexrosenkranz.com

## Overview

A personal website serving as a digital garden and curated directory of interests. The site prioritizes a minimal, typography-focused design with generous whitespace, optimized for readability and content discovery. It is not career-oriented but provides relevant links to professional profiles.

**Domain:** alexrosenkranz.com  
**Primary Framework:** Astro  
**Hosting:** Cloudflare Pages  
**Dynamic Runtime:** Cloudflare Workers

---

## Goals

1. **Low-friction content sharing**: Make it easy to post small things (links, photos, products, code snippets, music) throughout the week
2. **Curated monthly digests**: Roll up posts into monthly summaries, with support for both automatic and manual curation
3. **Long-form writing**: Support for occasional blog posts that can grow in frequency over time
4. **Persistent now-playing**: Display current Spotify listening status across all pages
5. **Minimal maintenance**: Static-first architecture with markdown-based content authoring
6. **Full observability**: Comprehensive Datadog instrumentation for monitoring and analytics

## Content Architecture

### Content Types

All content lives in Astro's `src/content/` directory using content collections.

#### 1. Posts (Quick Items)

Small, discrete items posted throughout the week.

```
src/content/posts/
в”ңв”Җв”Җ 2025-01-12-interesting-article.md
в”ңв”Җв”Җ 2025-01-13-cool-product.md
в””в”Җв”Җ 2025-01-14-code-snippet.md
```

**Frontmatter schema:**

```yaml
---
title: "Interesting Article About Web Performance"
publishedAt: 2025-01-12T10:30:00Z
tags:
  - article
  - web-dev
url: "https://example.com/article"  # optional, for link posts
image: "./photo.jpg"                 # optional, for photo posts
draft: false
---

Optional body content for additional commentary.
```

**Supported tags** (extensible):
- `article` - Links to external articles
- `photo` - Images with optional caption
- `product` - Physical or digital products
- `code` - Code snippets or repos
- `music` - Albums, songs, playlists
- `video` - YouTube, Vimeo, etc.
- `quote` - Text quotes
- `note` - Standalone thoughts
- `website` - Interesting websites
- `app` - Mobile or desktop applications
- `movie` - Films
- `tv` - TV shows

#### 2. Digests (Monthly Rollups)

Curated collections of posts, generated monthly.

```
src/content/digests/
в”ңв”Җв”Җ 2025-w02.md
в””в”Җв”Җ 2025-w03.md
```

**Frontmatter schema:**

```yaml

excludedPosts:               # optional, posts to exclude from auto-digest
  - "2025-01-07-draft-thought"
```

## Design System

### Design Principles

1. **Typography-first**: Let the text breathe; the content is the design
2. **Generous whitespace**: Ample padding and margins throughout
3. **Minimal chrome**: No unnecessary UI elements or decorations
4. **Dark mode support**: Default to system preference with manual toggle
5. **Responsive but not flashy**: Works well on all devices without complex breakpoints

### Design References

- **Brian Lovin** (brianlovin.com): Clean hierarchy, card-based project grid, app-like navigation
- **Carl Barenbrug** (carlbarenbrug.com): Editorial typography, generous line-height, moody dark aesthetic

### Cloudflare Workers

#### 1. Spotify Now Playing (`/api/spotify/now-playing`)

Proxies requests to Spotify API to avoid exposing credentials client-side.

**Endpoint:** `GET /api/spotify/now-playing`

**Response:**

```json
{
  "isPlaying": true,
  "title": "Song Title",
  "artist": "Artist Name",
  "album": "Album Name",
  "albumArt": "https://...",
  "songUrl": "https://open.spotify.com/track/...",
  "progress": 45000,
  "duration": 210000
}
```

If nothing is playing, returns the most recently played track with `isPlaying: false`.

**Client-side polling:**

The `NowPlaying` component polls the endpoint continuously while the page is open:

```typescript
// Poll every 30 seconds when not playing, every 10 seconds when playing
const POLL_INTERVAL_IDLE = 30000;
const POLL_INTERVAL_ACTIVE = 10000;

// Use visibilitychange API to pause polling when tab is hidden
document.addEventListener('visibilitychange', () => {
  if (document.hidden) {
    clearInterval(pollInterval);
  } else {
    poll(); // Immediate poll on tab focus
    startPolling();
  }
});
```

**Environment variables (Cloudflare secrets):**
- `SPOTIFY_CLIENT_ID`
- `SPOTIFY_CLIENT_SECRET`
- `SPOTIFY_REFRESH_TOKEN`

#### 2. Digest Generator (Scheduled Worker)

Runs monthly (Sunday midnight) to generate digest files.

**Logic:**
1. Query all posts from the past week
2. Generate markdown file with frontmatter
3. Commit to repository via GitHub API (or store in KV and trigger rebuild)

**Environment variables:**
- `GITHUB_TOKEN`
- `GITHUB_REPO`

```typescript
// In Cloudflare Worker
async function logToDatadog(message: string, context: Record<string, unknown>) {
  await fetch('https://http-intake.logs.datadoghq.com/api/v2/logs', {

```

## Appendix: Spotify OAuth Setup

To get a `SPOTIFY_REFRESH_TOKEN`:

1. Create app at https://developer.spotify.com/dashboard
2. Set redirect URI to `http://localhost:3000/callback`
3. Request authorization:
   ```
   https://accounts.spotify.com/authorize?client_id=<CLIENT_ID>&response_type=code&redirect_uri=http://localhost:3000/callback&scope=user-read-currently-playing%20user-read-recently-played
   ```
4. Exchange code for tokens:
   ```bash
   curl -X POST https://accounts.spotify.com/api/token \
     -H "Content-Type: application/x-www-form-urlencoded" \
     -d "grant_type=authorization_code&code=<CODE>&redirect_uri=http://localhost:3000/callback&client_id=<CLIENT_ID>&client_secret=<CLIENT_SECRET>"
   ```
5. Save the `refresh_token` (it doesn't expire)
